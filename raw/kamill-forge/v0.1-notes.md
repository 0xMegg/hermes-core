# Kamill Forge v0.1 Source Notes

Status: Source record for Phase 0 documentation and policy.
Collected: 2026-05-13

This file records the user-approved source decisions for Kamill Forge v0.1. It is source material, not active policy. Active guidance belongs in `policy/kamill-forge.md`; synthesized durable knowledge belongs in `wiki/pages/kamill-forge-overview.md`.

## Naming Decision

The user-approved name for the Hermes Experience Distillation Loop / governance layer is **Kamill Forge**.

For Phase 0, `hermes-core` remains the repository and operating-layer template name. Kamill Forge names a governance layer over Hermes' existing self-improvement mechanisms, not a replacement repository or parallel project identity.

## Scope

Kamill Forge is not a parallel automatic learning system. It is a governance layer over Hermes' existing self-improvement mechanisms:

- memory
- skills
- session_search
- curator

## Approved v0.1 Defaults

- Threshold-primary design.
- Script-only watchdog for quiet-day checks.
- Avoid false positives.
- No LLM calls on quiet days.
- Unanchored Discord candidates stay global/unanchored until the user retargets them.
- Codex proposes.
- Claude judges.
- Kamill/Hermes synthesizes in Korean.
- User decides.
- Staged proposal only before explicit commit approval.
- No automatic memory, skill, Core, or project modifications without explicit approval.
- User-originated Kamill/Hermes improvement ideas are included in the same flow as a distinct Kamill Improvement Lane.

## Phase 0 Boundary

Phase 0 is a paper-run introduction of the governance layer into this repository. It may document the design, policy boundaries, provenance, and verification expectations. It must not introduce automatic execution scaffolding.

Out of scope for Phase 0:

- Repository rename.
- Broad Hermes-to-Kamill renaming.
- Watchdog scripts.
- Cron jobs.
- Skills.
- Hooks.
- Curator changes.
- Automatic memory, skill, Core, or downstream project mutation.
